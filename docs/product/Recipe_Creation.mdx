# Recipe Creation

The **Recipe Creation** feature allows you to define the raw materials required to prepare or produce a menu item.
This helps the system automatically reduce ingredient stock during sales and calculate item cost based on inventory transactions.

---

# Example Scenario

## Menu Item

Before creating a recipe, you must first create the **Menu Item** in the Item Master.

Example:

| Menu Item |
| --------- |
| Pizza     |

| Cheese        |
| Toppings      |

# Understanding UOM and Conversion Factor

Sometimes materials are purchased in one unit but consumed in another unit.

Example:

* Flour is purchased in **KG**
* Recipe consumption is in **Grams (G)**

In this case, a **Conversion Factor** is required.

---

## Example — Flour

| Field             | Value |
| ----------------- | ----- |
| Purchase UOM      | KG    |
| Consumption UOM   | G     |
| Conversion Factor | 1000  |
| Recipe Usage      | 300 G |

### Explanation

* `1 KG = 1000 G`
* Therefore, the conversion factor is `1000`
* If one Pizza requires `300 G`, enter:

  * Consumption UOM = `G`
  * Quantity = `300`

---

## Example — Cheese

| Field             | Value |

## Example — Toppings

# Steps to Create a Recipe

## 1. Open Menu Master 

1. Select the menu item to produce.
2. Example:

   * Pizza

---

## 2. Select the Menu Item

* Navigate to **Recipe Management** from the ERP menu.

---

## 3. Add Material Items

1. Add all required material items.
2. Enter:

   * Material Item
   * UOM
   * Conversion Factor
   * Required Quantity

# Cost Calculation Logic

## Initial Cost

Initially, item costing is taken from:

* `initial_cost` in Product Master

This cost is used in reports such as:

* Stock Balance Report
* Inventory Valuation

---

# Cost Update After GRN

After a **GRN (Goods Received Note)** is processed:

* The system updates the product cost using:

  * `last_net_cost`

This means future inventory valuation and recipe costing will use the latest purchase cost from GRN transactions.

---

# Cost Flow Summary

| Stage      | Cost Source   |
| ---------- | ------------- |
| Before GRN | initial_cost  |
| After GRN  | last_net_cost |

---

# Stock Balance Report Behavior

## Before GRN

The stock balance report shows cost using:

* `initial_cost`

## After GRN

The stock balance report automatically uses:

* `last_net_cost`

This ensures inventory valuation reflects the latest purchase price.

---

# Important Notes

> Always ensure the correct UOM and Conversion Factor are configured before creating recipes.

> Incorrect conversion factors may cause inaccurate stock deduction and costing.

> Recipes are used during sales processing to automatically reduce material stock.
